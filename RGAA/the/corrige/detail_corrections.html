Exercice 1 : La structure de la page

Structure HTML5

  • Un élément <header> est ajouté autour de la zone d'en-tête, incluant ou non la ou les zones de navigation et la zone de recherche.
  • Des éléments <nav> sont ajoutés autour du ou des lien(s) d'accès rapide et autour du menu de navigation (en version mobile, la balise <nav> intègre le bouton burger qui permet d'afficher le menu).
  • Un élément <main> est ajouté autour de la zone de contenu principal.
  • Optionnellement, un élément <aside> est ajouté autour de la zone des réseaux sociaux.
  • Un élément <footer> est ajouté autour de la zone de pied de page.

Éléments de navigation interne

Landmarks :

  • Un role="banner" est défini sur l'élément <header>.
  • Un role="search" est défini sur la <div> ou sur le <form> contenant le champ de recherche.
  • Des role="navigation" sont définis sur les éléments <nav>.
    Pas obligatoire mais recommandé : des propriétés aria-label sont également définies sur ces éléments afin de les nommer. Par exemple : aria-label="Accès rapide" et aria-label="Menu principal".
  • Un role="main" est défini sur l'élément <main>.
  • Un role="complementary" est défini sur l'élément <aside> s'il existe.
  • Un role="contentinfo" est défini sur l'élément <footer>.

Liens d'accès rapide :

  • Au moins un lien (ancre) est ajouté au début de la page dans le code HTML et mène vers le contenu principal.

Exercice 2 : le contenu (partie 1)

Réparer les éléments du contenu

Titres :

  • Un <h1> est ajouté sur le texte "Découvrez les saveurs du thé".
  • Les titres de niveau 3 dans le pied de page sont changés en <h2>.
  • Un <h1> est ajouté sur le texte "Étymologie" dans la modale.

Listes :

  • Une liste <ul>/<li> est utilisée pour structurer les liens du menu de navigation.
  • Une liste <ul>/<li> est utilisée pour structurer le texte sous "Quelques thés parfumés célèbres".

Tableau :

  • Le texte "Production des 10 principaux pays producteurs en 2017" est déplacé directement après la balise <table> ouvrante, dans un élément <caption>.
  • Des éléments <th> remplacent les <td> de la première ligne et optionnellement de la première colonne du tableau.
    Si des éléments <th> sont utilisés dans la première colonne, alors des attributs scope sont ajoutés sur tous les éléments <th> du tableau (avec pour valeurs : col dans la première ligne, row dans la première colonne).

Exercice 3 : le contenu (partie 2)

Réparer les éléments du contenu

Images :

  • Une propriété aria-hidden="true" est ajoutée sur l'icône <svg> dans le bouton "Espace privé".
  • La structure <figure>/<figcaption> avec ses propriétés ARIA associées est utilisée autour de l'image "theier.jpg" et sa légende.
  • Un attribut alt="" est ajouté sur l'image "salon.jpg" dans le pied de page.

Liens :

  • Sur le lien-logo dans l'en-tête :
    • L'attribut alt de l'image est vide ;
    • Le title est modifié de façon à reprendre au moins le contenu du lien (par exemple : title="Le Thé : Accueil"). Le title pourrait aussi être supprimé (l'utilisateur comprend sans cette précision).
  • Dans la zone des réseaux sociaux, les intitulés des liens sont définis par leurs contenus, des textes positionnés hors écran.

Exercice 4 : Présentation

Réparer les erreurs liées à la présentation des contenus

Présence et ordre de lecture des contenus sans mise en forme CSS :

  • Le bandeau des cookies est déplacé en tout début de page dans le code HTML ;
  • Dans le formulaire "Catalogue", le bouton de soumission est déplacé après la case à cocher dans le code HTML ;
  • Dans le pied de page, les indications "Nouveau" sont intégrées directement en texte dans le contenu des liens et non au moyen de la propriété CSS content.

Visibilité des prises de focus :

  • Dans le fichier styles.css, la règle qui annulait l'outline sur les boutons est supprimée (sélecteur .tm-btn:focus).

Textes cachés :

  • Bouton "Menu" (version mobile) :
    • La propriété style="display: none;" est supprimée du <span> qui contient l'intitulé.
      Le texte est caché visuellement grâce à un positionnement hors écran (classe .sr-only dans styles.css).
    • La propriété aria-hidden="true" est conservée sur l'icône car elle ne doit pas être restituée.
  • Bouton "Espace privé" : La propriété aria-hidden="true" est supprimée du <span> qui contient l'intitulé.

Espacements des textes :

Dans fichier styles.css :

  • Menu de navigation : les propriétés overflow et height au minimum sont supprimées sur les liens (sélecteur .nav-list .nav-link) ;
  • Sous-menu "Espace privé" : la propriété height est supprimée sur les éléments de la liste (sélecteur .account-menu li) ;

Informations transmises par la couleur :

  • Lien actif du menu de navigation ("Accueil") :
    • Un changement de forme (soulignement) est ajouté sur le lien ;
    • Un title est ajouté sur le lien : title="Accueil - page active" .
    • Remarque : en réalité, l'utilisation d'un title remplit la condition du changement de forme puisqu'il déclenche l'affichage d'une tooltip. Avec la limitation cependant que cette tooltip n'apparaît pas au focus pour les personnes qui naviguent au clavier.

Exercice 5 : les formulaires

Réparer les formulaires

Étiquettes

  • Pour tous les champs du formulaire "Catalogue", des étiquettes sont définies au moyen d'éléments <label for="id"> ;
  • Pour le champ de recherche, l'étiquette est définie au moyen d'un attribut title (nom accessible) qui reprend le texte du placeholder (nom visible).

Regroupements

  • Le texte "Je souhaite recevoir un échantillon ..." et les boutons radio sont regroupés dans un élément <fieldset>.
  • Un élément <legend> remplace le <p> autour du texte "Je souhaite recevoir un échantillon...".

Aides à la saisie

Pour le formulaire "Catalogue" :

  • Champs obligatoires (tous les champs sauf les boutons radio et la case à cocher) :
    Des astérisques sont ajoutés dans les étiquettes des champs et un texte "* Champs obligatoires" est ajouté avant le début du formulaire ;
  • Formats attendus :
    Un texte "Format attendu : nom@domaine.com" visible est ajoutée dans un paragraphe à proximité du champ "Adresse e-mail". Une propriété aria-describedby est utilisée sur le champ pour lier le texte.
  • Messages d'aide :
    Une propriété aria-describedby est utilisée sur le champ "Adresse e-mail" pour y lier le texte "Nous nous engageons à ne pas communiquer pas votre adresse e-mail à des tiers".
  • Messages d'erreur :
    Des propriétés aria-describedby sont ajoutées sur champs pour les liers aux messages d'erreur associés.

Intitulé des boutons :

  • Un attribut title est ajouté sur le bouton de validation afin de préciser sa fonction.

Remplissage automatique des champs :

  • Des attributs autocomplete sont ajoutés sur les champs qui collectent des données personnelles :
    • Nom : autocomplete="family-name"
    • Prénom : autocomplete="given-name"
    • Adresse : autocomplete="street-address"
    • Code postal : autocomplete="postal-code"
    • Ville : autocomplete="address-level2"
    • Adresse e-mail : autocomplete="email"

Exercice 6 : JavaScript et ARIA.

Disclosure

Pour le bouton "Espace privé" :

  • Un <button> est utilisé à la place du <a> ;
  • Les propriétés aria-expanded et optionnellement aria-controls sont ajoutées ;
  • Dans le fichier script.js, la valeur de la propriété aria-expanded est mise à jour dans la fonction toggleAccountMenu.

Ordre de tabulation

Le contenu du sous-menu "Espace privé" est positionné dans le code source avant le bouton qui gère sa visibilité. Afin de garantir un ordre de tabulation logique, le contenu du sous-menu est déplacé après le bouton "espace privé" dans le code source.

L'affichage du sous-menu "Espace privé" est géré au moyen de la propriété CSS display au lieu de opacity, évitant la tabulation sur les éléments non visibles du sous-menu lorsque le celui-ci est fermé.

Message de statut

Une propriété role="status" est ajoutée sur l'élément qui contient la notification de prise en compte des préférences de l'utilisateur concernant les cookies (#cookiesConfirm).

Formulaire

Focus

  • Après soumission du formulaire avec des erreurs, la méthode JS focus() est utilisée pour déplacer le focus de l'utilisateur sur le premier champ qui contient une erreur de saisie : la correction est à faire dans le fichier script.js, dans la fonction verifyRequiredFields (sous le commentaire "Don't send form").
  • Après la soumission du formulaire complet, sans erreurs, la méthode JS focus() est utilisée pour déplacer le focus de l'utilisateur sur le texte de réussite "Votre demande a bien été reçue. Vous recevrez notre catalogue très prochainement" : la correction est à faire dans le fichier script.js, avant successCtn.focus(); en ajoutant une propriété tabindex="-1" au passage de texte visé par la reprise de focus.

Fenêtre modale

  • Les propriétés suivantes sont ajoutées sur l'élément #dialog-wrapper :
    • role="dialog"
    • aria-modal="true"
    • aria-labelledby="dialog-title"
  • Un attribut tabindex="-1" est ajouté sur le même élément afin de permettre sa prise de focus à l'ouverture de la modale ;
  • la méthode JS focus() est utilisée pour déplacer le focus de l'utilisateur :
    • Sur la modale elle-même (#dialog-wrapper) à son ouverture ;
    • Sur le bouton #btn-dialog à sa fermeture.
    La correction est à faire dans le fichier script.js, dans les fonctions openDialog et closeDialog.
